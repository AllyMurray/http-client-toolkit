The dashboard package provides a browser-based UI for monitoring your cache, dedup, and rate limit stores in real time. It works with all store backends (Memory, SQLite, DynamoDB) and auto-detects capabilities.

## Embedded Middleware

Mount the dashboard inside an existing HTTP server using `createDashboard()`. Pass in any combination of your store instances:

```typescript
import http from 'http';
import { createDashboard } from '@http-client-toolkit/dashboard';
import {
  InMemoryCacheStore,
  InMemoryDedupeStore,
  InMemoryRateLimitStore,
} from '@http-client-toolkit/store-memory';

const cacheStore = new InMemoryCacheStore();
const dedupeStore = new InMemoryDedupeStore();
const rateLimitStore = new InMemoryRateLimitStore();

const dashboard = createDashboard({
  cacheStore,
  dedupeStore,
  rateLimitStore,
});

const server = http.createServer(dashboard);
server.listen(4000, () => {
  console.log('Dashboard at http://localhost:4000');
});
```

The middleware signature is `(req, res, next?) => void`, so it works with plain `http.createServer`, Express, Connect, and similar frameworks.

## Standalone Server

For convenience, `startDashboard()` spins up its own HTTP server:

```typescript
import { startDashboard } from '@http-client-toolkit/dashboard';

const { server, close } = await startDashboard({
  cacheStore,
  dedupeStore,
  rateLimitStore,
  port: 4000,
  host: 'localhost',
});

// Later...
close();
```

## Configuration

### `DashboardOptions`

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `cacheStore` | `CacheStore` | — | Cache store instance to monitor |
| `dedupeStore` | `DedupeStore` | — | Dedup store instance to monitor |
| `rateLimitStore` | `RateLimitStore` | — | Rate limit store instance to monitor |
| `basePath` | `string` | `'/'` | URL prefix for the dashboard |
| `pollIntervalMs` | `number` | `5000` | How often the UI polls for updates (ms) |

At least one store must be provided.

### `StandaloneDashboardOptions`

Extends `DashboardOptions` with:

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `port` | `number` | `4000` | Port number for the standalone server |
| `host` | `string` | `'localhost'` | Host to bind to |

## Supported Backends

The dashboard uses runtime detection to identify which store implementation you've passed in and wraps it with a backend-specific adapter. No changes to your store code are required.

| Backend | Cache | Dedup | Rate Limit |
|---------|-------|-------|------------|
| **Memory** | Stats, list entries, view/delete/clear | Stats, list jobs, view job | Stats, list resources, view/reset/configure |
| **SQLite** | Stats, list entries, view/delete/clear | Stats, list jobs, view job | Stats, list resources, view/reset/configure |
| **DynamoDB** | Stats only | Stats only | Stats only |
| **Unknown** | Stats only | Stats only | Stats only |

:::note
DynamoDB and unknown store types fall back to a generic adapter with limited capabilities. The UI gracefully hides features that aren't available.
:::

## API Endpoints

The dashboard exposes a REST API that the React SPA consumes. You can also call these endpoints directly.

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/api/health` | Store types and capabilities |
| `GET` | `/api/stores` | Connected stores summary |
| `GET` | `/api/cache/stats` | Cache statistics |
| `GET` | `/api/cache/entries?page=0&limit=50` | List cache entries |
| `GET` | `/api/cache/entries/:hash` | Get a single cache entry |
| `DELETE` | `/api/cache/entries/:hash` | Delete a cache entry |
| `DELETE` | `/api/cache/entries` | Clear all cache entries |
| `GET` | `/api/dedup/stats` | Dedup statistics |
| `GET` | `/api/dedup/jobs?page=0&limit=50` | List dedup jobs |
| `GET` | `/api/dedup/jobs/:hash` | Get a single dedup job |
| `GET` | `/api/rate-limit/stats` | Rate limit statistics |
| `GET` | `/api/rate-limit/resources` | List rate-limited resources |
| `GET` | `/api/rate-limit/resources/:name` | Get resource status |
| `PUT` | `/api/rate-limit/resources/:name/config` | Update resource config |
| `POST` | `/api/rate-limit/resources/:name/reset` | Reset a resource counter |
